# MDReader — v0 Build Report

**Built:** 2026-04-17
**Status:** ✅ Complete

## What's Working

- Opens `.md` files via the native iOS Files browser (DocumentGroup)
- Creates new markdown documents from the launch screen
- Segmented "Preview / Edit" control in the navigation bar
- Preview tab renders markdown with inline images (WKWebView + marked.js)
- Edit tab is a plain monospaced text editor
- Changes save back to the original file automatically on close/background (DocumentGroup default behavior)
- Dark mode works (system-adaptive CSS in the preview template)

## How to Run It

```bash
cd /Users/san/Projects/md-reader

# Regenerate project if you edit project.yml
xcodegen generate

# Build + run in Simulator
xcodebuild \
  -project MDReader.xcodeproj \
  -scheme MDReader \
  -destination 'platform=iOS Simulator,name=iPhone 16e' \
  build

# Or open in Xcode and press ⌘R
open MDReader.xcodeproj
```

## Visual QA

**Final screenshot:** `.v0-swift/screenshots/final.png` (launch / document browser)

**Limitation discovered during v0 build:** iOS Simulator doesn't expose a clean CLI for driving in-app UI taps. AppleScript tap coordinates are unreliable and caused an accidental rotation during QA. The system-provided DocumentGroup browser (the first screen) was verified via screenshot. The inner `DocumentView` (Preview/Edit tabs) was not automatically screenshotted — it requires a 2-tap manual test:

1. Open the app in Simulator
2. Tap "Create Document" (or any existing `.md` file via Browse → On My iPhone)
3. Type some markdown in the Edit tab
4. Tap "Preview" — should render with images, code blocks, tables, etc.

This is flagged as a known /v0-swift skill limitation to improve in the next iteration.

## Key Decisions

- **DocumentGroup + FileDocument** over manual `UIDocumentPickerViewController` — native file browser, save/rename/recents come free
- **WKWebView + marked.js** over native `AttributedString(markdown:)` — only way to render inline images (`![alt](image.png)`)
- **Vendored marked.js (40KB)** over CDN load — works offline, no network required
- **Set WKWebView `baseURL` = document's parent directory** so relative image paths resolve
- **Skipped `CFBundleDocumentTypes` registration** for v0 — DocumentGroup with `readableContentTypes: [.plainText, .markdown]` handles picking `.md` files fine. Registering the app as a system-wide `.md` handler (so it shows in Files app's "Open in…" menu) is deferred to v1 — requires hand-written Info.plist.
- **Stock Apple HIG** for all chrome — system tint, system fonts, no custom toolbar styling
- **Removed `.ignoresSafeArea(.keyboard)`** from EditView — it would hide text behind the keyboard, opposite of what we want

## Known Issues / Deferred to v1

- App isn't registered as a system `.md` handler (can't "Open in MDReader" from Files app). Fix: add hand-written Info.plist with `CFBundleDocumentTypes` array.
- No syntax highlighting in Edit mode (out of v0 scope)
- No search within a document (out of v0 scope)
- Large markdown files (>1 MB) may stutter in preview — marked.js is synchronous
- No explicit "Save" button — DocumentGroup saves automatically but users unfamiliar with that pattern may look for one
